Enhancing Biomedical Content Authoring Through Knowledge Representation

There is a plethora of biomedical documents available to read on the internet. However, a significant majority of these texts are unstructured and lack semantic metadata that gives context for search engines to retrieve more precise information. To ameliorate this problem, many semantic annotators have been developed such as NCBO and RYSannMD. Even so, these annotators still struggle to achieve optimal accuracy and speed metrics with their approaches to biomedical semantic content annotation. For our use case, we have introduced a knowledge graph based recommendation system approach that allows users to choose prominent semantic annotation recommended by the domain through a socio-technical and personalized approach. We have implemented this approach in "Semantically", a biomedical content semantic annotation platform to enhance semantic annotation of biomedical texts and embed it into the heading of documents for more accurate data retrieval.

This page is meant to present the current workflow and functionalities of the system, showing the diffrent approaches such as using a social-technical approach, personalized approach and a knowledge graph based recommendation system enhancement. Biomedical Semantic Annotation process + Social Technical Approach + Personalized Approach

This is the demo for the social-technical approach. This approach was implemented to enhance the process of biomedical semantic annotation by involving other users/domain experts in the semantic annotation process. By creating an enviroment in which users can seek for assistance from a community or other experts, the social-technical approach significantly improves the ease and accuracy of annotating by asking for help from qualified users with finding the best ontologies for embedding relevant context for a given term.

For demonstration purposes, the following videos show an example of how a user may engage in the semantic annotation process through a social-technical approach. The steps demonstrated are as follows:

Social-Technical Recommendation

  • Step 1:

    The user selects a specific annotation in their document: "virus-replication-cycle". This opens an option to ask a social technical question or personalized recommendation.

  • Step 2:

    A prompt will open where the user can select one of three questions for the selected term:

    • "Which ontology should I use?"

    • "What is the suitable ontology vocabulary?"

    • "Does this ontology best describe this terminology?"

    The user also provides a description of what exactly they need. Once submitted, the post is publicized in the Semantically Cafe Forum.

  • Step 3:

    Other domain experts and users respond by suggesting recommendations for ontologies. Users will also give themselves confidence_scores and explanations for their suggestions. In the videos featured below, two recommendations are made with different responses and confidence scores.

  • Step 4:

    After responses are submitted, the original user can select the same term again (virus-replication-cycle) and see the new recommendations after clicking on the "New Recommendation" link that appears.

  • Step 5:

    The user opens a prompt showing the responses given by the other domain experts. The user has the option to choose between accepting or rejecting the recommendation.

    • If accepted, the user is prompted to give a rating from 1 to 5. The ontology and definition is then updated according to the accepted recommendation.

    • If rejected, the prompt is closed and the original ontology of the term remains unchanged.

Biomedical Semantic Annotation process + Social Technical Approach + Personalized Approach + Knowledge Graph Recommendation Tab

In this section, we provide videos/images of the demonstration for the Knowledge Graph Recommendation System process. The Knowledge Graph is meant to further enhance the process by providing a search functionality that enables users to look for the best recommendations, domain experts and best suggestions for ontologies. A knowledge graph with detailed information of the best recommendations retrieved is generated to provide a clear visualization and representation of annotations/recommendations.

UPDATE: After some feedback, the scoring system has been reevaluated to account for the potential bias that can result from a self-evaluation score. The values now have weights, with confidence scores having the lowest weight (0.20) to the wilson score and rating score (0.40). This helps to ensure that confidence scores, which can be overestimated/underestimated by domain experts, have less sway over which recommendations are prioritized over others.
